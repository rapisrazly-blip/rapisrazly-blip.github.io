# cc-lp-v2 — AIでLPを作るツール

AIと対話するだけで、プロ品質のランディングページ（LP）を作れるツールです。
最後はFigmaの編集可能なデザインにもできます。

プログラミングの知識は不要です。AIがコードを書きます。あなたは「どんなLPを作りたいか」を伝えるだけです。

---

## できること

1. ヒアリング（AIがあなたのビジネスについて質問）
2. **AIが参考LPを Web から探してきて提案**（自分で探す手間なし）
3. デザインの方向性を提案
4. コピー（文言）を提案
5. **画像はAIが自動でいい感じのものを配置**（ロゴ・顔写真・ヒーロー写真）→ 手元の画像があれば渡すだけでOK
6. HTMLでLPを生成 → ブラウザで自動プレビュー
7. **PC表示もスマホ表示も両対応**（レスポンシブ対応）
8. 「この画像を差し替えて」などの修正ループ
9. Figma用のファイルに書き出し → Figmaにインポートして編集可能に

---

## 始める前に必要なもの

以下の2つのソフトをインストールする必要があります。
既にインストール済みなら、[「始め方」](#始め方) まで読み飛ばしてください。

1. **Node.js** — このツールの裏側で動くプログラム
2. **Claude Code** — AIと対話するためのコマンドラインツール

---

## Step 1: Node.js をインストールする

Node.js は、このツールの裏側で動くプログラムです。
一度インストールすれば、あとは意識する必要はありません。

### ダウンロード

1. https://nodejs.org を開く
2. **左側の「LTS」と書かれたボタン** をクリックしてダウンロード
   （「LTS」は安定版という意味です。右側の「Current」は選ばないでください）

### Mac の場合

1. ダウンロードした `.pkg` ファイルをダブルクリック
2. 「続ける」→「続ける」→「同意する」→「インストール」と進む
3. パスワードを求められたら、Macのログインパスワードを入力
4. 「インストールが完了しました」と表示されたら完了

### Windows の場合

1. ダウンロードした `.msi` ファイルをダブルクリック
2. 「Next」→「I accept the terms...」にチェック →「Next」→「Next」→「Next」→「Install」と進む
3. 「ユーザーアカウント制御」のダイアログが出たら「はい」をクリック
4. 「Finish」をクリックして完了

### インストールの確認（やらなくても大丈夫です）

確認したい場合は:
- Mac: 「ターミナル」アプリを開いて `node -v` と入力して Enter
- Windows: 「コマンドプロンプト」を開いて `node -v` と入力して Enter

`v20.xx.x` のようなバージョン番号が表示されれば成功です。

## Step 2: Claude Code をインストールする

Claude Code は、Anthropic社（AI「Claude」を作っている会社）が提供するコマンドラインツールです。
ターミナル上で AI と対話しながら、コードやファイルを自動で作ってもらえます。

### 公式ドキュメント

最新のインストール手順は公式ドキュメントが正確です:
https://docs.anthropic.com/en/docs/claude-code

以下は 2026年4月時点での代表的な手順です。

### Mac の場合

1. 「ターミナル」アプリを開く
   （Finder → アプリケーション → ユーティリティ → ターミナル.app）
2. 以下のコマンドをコピーしてターミナルに貼り付け、Enter を押す:

   ```bash
   curl -fsSL https://claude.ai/install.sh | bash
   ```

3. インストールが終わったら、ターミナルで以下を実行して確認:

   ```bash
   claude --version
   ```

   バージョン番号が表示されれば成功です。

### Windows の場合

Windows でのインストール方法は環境によって変わります。
以下の公式手順を参照してください:
https://docs.anthropic.com/en/docs/claude-code

> うまく入らない場合は、「Windows Claude Code インストール」で検索すると最新の手順が見つかります。

### 初回ログイン

1. ターミナルで `claude` と入力して Enter
2. 初回はログインを求められます。画面の指示に従ってブラウザで Anthropic アカウントでログインしてください
3. 「Welcome to Claude Code」のような表示が出れば完了です

## 始め方

### 1. このフォルダを手元に用意する

共有された **Google Drive のリンク** をブラウザで開き、そこから `cc-lp-v2` フォルダをパソコンに保存します。

**手順:**

1. 共有された Google Drive のリンクをブラウザで開く
2. 画面上の `cc-lp-v2` フォルダを **右クリック** →「**ダウンロード**」を選ぶ
   - Google Drive が自動で zip ファイル（例: `cc-lp-v2-XXXXXXXX.zip`）にまとめてダウンロードしてくれます
   - ダウンロードには少し時間がかかることがあります（数秒〜十数秒）
3. ダウンロードが終わったら、zip ファイルを **ダブルクリックして解凍**
   - Mac: 自動で解凍され、同じ場所に `cc-lp-v2` フォルダができます
   - Windows: 右クリック →「すべて展開」で同じ場所に展開されます
4. 出てきた `cc-lp-v2` フォルダを、**分かりやすい場所**に置いてください
   - おすすめ: **デスクトップ** または **ドキュメント** フォルダ
   - ダウンロードフォルダに置いたままでもOKですが、あとで見つけやすい場所に動かすと楽です

**注意:**
- フォルダ名の `cc-lp-v2` は **変えないでください**（そのままの名前で使います）
- 解凍前の zip ファイルは、解凍が終わったら削除して大丈夫です
- Google Drive 側のフォルダはそのまま残しておいてOKです（ダウンロード＝コピーなので元は消えません）

> Google Drive のデスクトップアプリを使っている方へ: 「マイドライブへのショートカット」として追加しただけではローカルに実体が無いため、一度「ダウンロード」してローカルに実体を置くのが確実です。

### 2. ターミナルで cc-lp-v2 フォルダに移動する

#### Mac の場合

1. 「ターミナル」アプリを開く
2. ターミナルに `cd ` と入力する（`cd` のあとに **半角スペース** を1つ）
3. Finder で `cc-lp-v2` フォルダを見つけ、**フォルダごとターミナルにドラッグ＆ドロップ**
   → 自動でフォルダのパスが入力されます
4. Enter を押す

> 例えば以下のような表示になります:
> ```
> cd /Users/あなたの名前/Downloads/cc-lp-v2
> ```

#### Windows の場合

1. エクスプローラーで `cc-lp-v2` フォルダを開く
2. 画面上部の **アドレスバー**（パスが表示されている部分）をクリックし、`cmd` と入力して Enter
   → そのフォルダで「コマンドプロンプト」が開きます

### 3. Claude Code を起動する

ターミナルで以下を入力して Enter:

```bash
claude
```

Claude Code が起動します。

### 4. 「LPを作りたい」と送る

チャット欄に以下を入力して、送信:

```
LPを作りたい
```

AIがヒアリングを始めます。**あとはAIの案内に従うだけです。**

## 使い方の流れ

Claude Code が起動したら、以下の流れで進みます。画面の指示に従って答えていくだけでOKです。

### 1. ヒアリング
AIがあなたのビジネスについて質問します:
- 何を売っているか
- 誰に向けたサービスか
- LPの目的（申し込み？問い合わせ？LINE登録？）
- 強み・差別化ポイント
- ロゴ画像があれば（この時点ではロゴだけ。他の画像は後でまとめて聞かれます）

### 2. AIが参考LPを探してくれる
ビジネスのヒアリングが終わると、**AIが Web 検索であなたの業界・ターゲットに合った参考LPを3つくらい探して提案してくれます。**
- 提案された中から気に入ったものを選ぶだけでOK
- 「このサイトみたいなデザインがいい」というURLを自分で持っていれば、追加で渡してもOK
- 参考LPの選定がLPの品質を大きく左右します。気に入ったものが見つかるまで「もっと別の雰囲気で探して」と伝え直せます

### 3. デザインの方向性を確認
AIが「こんなデザインでいきます」と提案してきます。
OKなら「OK」と返すだけ。違う方向がよければ希望を伝えてください。

### 4. コピー（文言）を確認
AIが各セクションのキャッチコピー・本文を提案します。
OKならそのまま進み、直したい箇所があれば伝えてください。

### 5. 画像素材を渡す（任意） ← 渡さなくてもOK。AIが自動で入れます
構成とコピーが確定したタイミングで、**AIが「このLPにはこういう画像があると効果的です」とセクション別にチェックリストで教えてくれます。**
- 例:
  - ファーストビューのメインビジュアル
  - 「解決策」セクションで見せたいスクショ
  - 「実績」セクションの数字グラフ
  - 「お客様の声」用の顔写真

**渡さなかった画像は AI が自動で入れます:**

| 画像の種類 | 自動補完の中身 |
|---|---|
| ロゴ | サービス名の頭文字を使ったシンプルロゴをSVGで自動生成 |
| 顔写真（お客様の声） | pravatar.cc からランダムな顔写真 |
| ヒーロー・商品・ビジネス写真 | **AIが Unsplash を検索して、ビジネスに合う写真を自動で配置** |
| 装飾イラスト | unDraw のフリーイラスト |

- ある画像だけチャットにドラッグ＆ドロップで渡してもOK
- **全部「このまま進めて」でOK**。画像も含めて AI が一気に仕上げてくれます
- 気に入らない画像は、ステップ7（修正ループ）で差し替え可能

### 6. HTMLプレビューで確認
AIがLPをHTMLで生成し、**ブラウザで自動的に開きます。**
出来栄えを確認してください。

**スマホ表示も必ず確認してください:**
- Chrome / Safari / Edge どのブラウザでも、画面を右クリック → 「検証」または「開発者ツール」を開く
- 左上のスマホアイコン（デバイスツールバー）をクリックすると、スマホ表示に切り替わります
- スマホ画面で文字や画像が崩れていないか、ボタンが押しやすそうか確認
- 崩れていれば、ステップ7（修正ループ）で「スマホでここが崩れてる」と伝えればOK

### 7. 修正ループ（画像の差し替えもここで）
「ここの色を変えて」「このセクションを前に持ってきて」「この文言をもっと短く」など、
気になる点を自由に伝えてください。AIがその場で修正します。

**AIが自動補完した画像の差し替えもこのタイミングでOK:**
- 「FVの写真、別の雰囲気のものに変えて」→ AIが Unsplash で再検索
- 「この顔写真、別の人に変えて」→ AIがランダム再生成
- 「ロゴ、こっちを使って」→ 自社ロゴをドラッグ＆ドロップで差し替え
- 「この画像、自社の写真に変えて」→ 画像を添付すれば差し替え

**レスポンシブの微調整も気軽にどうぞ:**
- 「スマホだと見出しが大きすぎる」「スマホでカードが詰まって見づらい」など伝えればAIが調整します

満足するまで何度でもやり直しできます。

### 8. Figmaに書き出す
「OK」「完成」「Figmaに送ろう」と伝えると、以下の流れで進みます:

1. AI がFigma用ファイル（`lp/figma-ready.html`）を自動で作成
2. **AI がそのHTMLをチャット欄にコードブロックとして出力** → 右上の「Copy」ボタンでワンクリックコピー
   - ファイルが大きい場合は、AI が自動でクリップボードに直接コピーしてくれます（`pbcopy` コマンド使用）
3. Figma を開いて `html.to.design` プラグインを起動 → Editor タブに貼り付け → インポート

Figmaインポートの詳細手順もチャットで案内されるので、そのまま従えばOKです。

## よくある質問

### 基本

**Q: プログラミングの知識は必要ですか？**
A: 不要です。AIがコードを書きます。あなたは「どんなLPを作りたいか」を伝えるだけです。

**Q: 何個までLPを作れますか？**
A: 制限はありません。別のLPを作りたい場合も、もう一度「LPを作りたい」と話しかければOKです。
ただし **前のLPは上書きされる** ので、残しておきたい場合は `lp/index.html` を別名でコピー・保存しておいてください。

**Q: 途中でClaude Codeを閉じてしまったらどうなりますか？**
A: もう一度ターミナルで `cc-lp-v2` フォルダに入り、`claude` で起動してください。
そのあと「さっきの続きから」と伝えれば、AIが状況を確認して再開してくれます。

### Google Drive について

**Q: Google Drive 上のフォルダを直接編集してもいいですか？**
A: **いいえ。Google Drive 上のファイルは編集しないでください。**
Google Drive はあくまで「配布用の置き場所」です。作業するときは必ず、
**一度ローカルPC（デスクトップ等）にダウンロード** してから、ダウンロードしたフォルダで Claude Code を起動してください。
Google Drive 上で直接いじると、Claude Code がファイルを見つけられずエラーになります。

**Q: アップデート版のフォルダが共有されたら、どうすればいいですか？**
A: 新しい `cc-lp-v2` フォルダを改めてダウンロードし直してください。
ただし、**あなたが作った LP のデータは手元にしか残っていません**（Google Drive 側には反映されません）。そのため:
- `lp/index.html`（今作っているLP）と `lp/assets/`（渡した画像）を、**必ず別の場所にコピーして退避** してから古いフォルダを置き換えてください
- 退避したファイルを、新しい `cc-lp-v2/lp/` の中に戻せば、続きから編集できます

**Q: Google Drive のデスクトップアプリで同期しているフォルダを直接使ってもいいですか？**
A: 基本的に **おすすめしません**。同期フォルダだと、ファイルの書き込みと Google Drive 側の同期がぶつかって不具合が起きる場合があります。必ず一度 **ローカルPC（デスクトップ等の通常フォルダ）にコピーしてから** 使ってください。

### 画像について

**Q: 画像を1枚も用意できないんですが、LPは作れますか？**
A: はい、作れます。画像素材が何もなくても、AIが自動で適切な画像を入れてくれます:
- ロゴ → サービス名の頭文字を使ったシンプルロゴをSVGで自動生成
- 顔写真 → pravatar.cc のランダム顔写真
- ヒーロー写真や商品写真 → Unsplash（商用フリー画像サイト）から AI が検索して選定
- 装飾イラスト → unDraw のフリーイラスト

気に入らない画像は、プレビューを見てから「この写真、別のに変えて」と伝えれば差し替えできます。

**Q: AI が自動で入れた画像は商用利用してOKですか？**
A: 基本的にOKです。使っているのは商用フリーのサービス（Unsplash, pravatar.cc, unDraw）だけです。
ただし本番で使う場合は、**ロゴと顔写真は最終的に本物（自社のロゴ、実際のお客様の顔写真）に差し替えることを強く推奨します。**
ヒーロー写真などは Unsplash のままでも問題ありません。

**Q: どんな画像形式が使えますか？**
A: PNG, JPG, GIF, SVG, WebP が使えます。

**Q: 画像はどうやって渡せばいいですか？**
A: Claude Code のチャット欄に画像ファイルをドラッグ＆ドロップするのが一番カンタンです。
うまくいかない場合は、`lp/assets/` フォルダに画像を直接置いてから、AIに「lp/assets/ に画像を入れました」と伝えてください。

**Q: 画像が多すぎると遅くなりますか？**
A: 最終的にFigmaに入れる際、全ての画像がファイルに埋め込まれるため、画像を多く使うとファイルサイズが大きくなります。
大きな画像（5MB以上）は事前にリサイズしておくことをおすすめします。

### スマホ表示（レスポンシブ）について

**Q: スマホでも表示できますか？**
A: はい、**自動的にスマホ対応されます**。PC表示とスマホ表示の2パターンで最適化され、画面幅 768px で切り替わります。

**Q: スマホで表示を確認する方法は？**
A: ブラウザの開発者ツールでカンタンにチェックできます:
- Mac/Windows共通: HTMLを開いている画面で右クリック → 「検証」または「開発者ツール」
- 開いたパネルの左上にある**スマホアイコン（デバイスツールバー）**をクリック
- 画面上部で iPhone や Android など機種を選べます
- 実機のスマホで確認したい場合は、**同じWi-Fi環境なら**PCのIPアドレスでアクセスできますが、初心者の方はブラウザの開発者ツールで十分です

**Q: スマホで表示が崩れていました**
A: AIに「スマホで〇〇セクションが崩れている」「スマホで見出しが大きすぎる」のように伝えれば修正してくれます。
どこが崩れているかスクショで送るとより早く伝わります。

### Figma書き出しについて

**Q: Figmaに貼り付けたら画像が欠損していました**
A: 貼り付けたファイルが `lp/index.html` ではなく **`lp/figma-ready.html`** になっているか確認してください。
`figma-ready.html` の方が画像を埋め込んだ「Figma用」ファイルです。

**Q: Figmaプラグインで「too large」と言われます**
A: 画像を減らすか、大きな画像を小さくリサイズしてから再度 AI に「Figma用ファイルを作り直して」と伝えてください。

**Q: アニメーションはFigmaに取り込んだらどうなりますか？**
A: Figmaはアニメーションの動きそのものを再現できません（Figma は動画ツールではなく静的なデザインツールなので）。
そのため **最終的な "止まっている状態"** として取り込まれます:
- **ホバーアニメーション**（マウスを乗せたときに浮き上がる等）→ 何もしていない通常の状態で取り込まれます。問題ありません
- **ヒーローの入場アニメーション**（ページを開いた瞬間にふわっと表示されるもの）→ アニメーションが終わった後の状態（完全に表示された状態）で取り込まれます。問題ありません
- **スクロールに合わせて出てくるアニメーション**（下にスクロールすると要素がふわっと現れるタイプ）→ **このプロジェクトでは使わないルールになっています**。理由は、Figmaにはこの仕組みを動かすためのブラウザ機能がなく、「透明のまま取り込まれて要素が消えたように見える」不具合が起きるためです。

念のため、`Figma用ファイル作成` を実行するときに、万が一こういう隠し要素が残っていても **強制的に表示させるルール** が自動で足されるようになっています。なので普通に使う分には心配いりません。

**Q: Figmaに取り込んだら、アニメーションしていた部分がズレたり、消えてしまっているように見えます**
A: LPにスクロール出現系のアニメーション（`.fade-in` や `data-aos` など）が残っていた可能性があります。
AIに「`lp/index.html` からスクロール出現系のアニメーションを全部外して、Figma用ファイルを作り直してください」と伝えてください。
